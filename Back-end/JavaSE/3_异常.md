## 异常处理机制
### try-catch-finally
```java
//finally可以没有
try{
可能会出现异常的代码
}catch（Exception e）{
异常发生时，将异常封装成对象e后传入
没有发生异常，catch里的不执行
}finally{
不管异常发没发生，都会执行。
通常放释放资源的代码
}
//如果finally有return，在catch里的return不返回但会执行；要返回或者抛出异常，先得执行finally里的内容
```
### thows
默认处理机制thows，不断向父类抛，抛到最后jvm报错

## 1.MySQL 中的数据排序是怎么实现的？
### 索引
1. order by 的字段正好有索引
2. 所有字段排序的方向一致
3. 联合索引要符合最左前缀原则
4. 索引要大量回表走全表扫描+filesort
5. 优先选择覆盖索引
### filesort
1. 索引用不了走filesort
2. 用sort_buffer内存缓冲区排序，数据量小于缓冲区在内存执行，不然在磁盘中做排序
3. 内存排序分：单路排序和双路排序   磁盘排序为：归并排序
4. 单路排序：查所有select的字段排序，排完直接返回，查一次
5. 双路排序：只查排序字段和行id排序，排完再去查其他字段返回，查两次
6. max_length_for_sort_data单路/双路排序切换阈值
7. 归并排序：超过sort_buffer_size,分成多个文件做快排，最后合成一个有序大文件

```sql
EXPLAIN SELECT * FROM user ORDER BY age;
-- 如果有  Extra: Using filesort 说明filesort排序

```
## 2.详细描述一条 SQL 语句在 MySQL 中的执行过程
1. 客户端与MySQL建立TCP连接，连接器负责连接和身份验证
2. 分析器对sql做词法分析拆成token，语法分析判断是否符合语法规则，生成语法树
```sql
SELECT id, name FROM user WHERE age > 18;
-- → 拆成：
-- SELECT（关键字）
-- id（列名）
-- FROM
-- user（表名）
-- WHERE
-- age > 18（条件）
```
3. 优化器决定这条 SQL 怎么执行最快生成执行计划
4. 执行器按照计划调用存储引擎接口返回结果

## 3.MySQL 的索引类型有哪些？
### 按数据结构分
1. B+树索引：最常用
2. 哈希索引：不支持范围查询，排序。本身等值查询
3. 全文索引
4. 空间索引
### 按InnoDB存储方式看
1. 聚簇索引：主键索引，叶子节点存完整的行数据，一张表就一个主键索引
2. 非聚簇索引：二级索引，叶子节点存索引字段和主键值，查完二级索引拿到主键值还得去聚簇索引查一遍，这个过程叫回表
### 按索引性质看
1. 主键索引：唯一非空，InnoDB的主键索引就是聚簇索引
2. 唯一索引：保证列值不重复，可有多个NULL
3. 普通索引：没有唯一约束
4. 全文索引：文本搜索用
5. 空间索引：GIS数据用
6. 联合索引和最左前缀原则：比如联合索引（a,b,c），B+树按a排序，再按b,c。只要不是顺序，就失效。

## 4.MySQL 的 Change Buffer 是什么？它有什么作用？
1. 是 InnoDB 用来缓存“非唯一二级索引页的变更操作（insert,delete,update）”的一块内存区域
2. 用于延迟修改索引页，从而减少随机磁盘 I/O，
3. 索引页不在Buffer Pool，它不去磁盘都索引页，而是将变更记录放一起，当索引页被访问或后台触发时再合并
4. Change Buffer属于Buffer Pool的一部分
5. 适用于写多、读少、非唯一二级索引、多随机写 的业务，如日志、消息队列、流水、批量导入等
